
# Alert Triage with Threat Intelligence

## Overview
Alert triage is an important activity in a Security Operations Center (SOC). Analysts review alerts generated by monitoring systems, determine their severity, and validate indicators of compromise (IOCs) using threat intelligence sources.

**Tools Used**

- Wazuh SIEM
- VirusTotal
- AlienVault OTX

## 1. Alert Triage Simulation

A simulated alert indicating **suspicious PowerShell execution** was analyzed in the Wazuh dashboard. PowerShell activity is often associated with malicious scripts or attacker post-exploitation techniques.

The alert was reviewed to identify the affected system, the source IP address, and the priority level.

### Alert Details

| Alert ID | Description | Source IP | Priority | Status |
|----------|-------------|-----------|----------|--------|
| 004 | PowerShell Execution | 192.168.1.101 | High | Open |

The alert was classified as **high priority** because suspicious PowerShell execution may indicate malicious script activity or unauthorized command execution.

## 2. IOC Validation

To validate the alert, the associated IP address was checked against external threat intelligence platforms.

### Threat Intelligence Sources

- **VirusTotal**
- **AlienVault Open Threat Exchange (OTX)**

The IP address **192.168.1.101** was searched in both platforms to determine whether it had been associated with malicious activity.

Since this IP address belongs to a private lab environment, no malicious reputation data was found.


## 3. Threat Intelligence Findings

Both VirusTotal and AlienVault OTX returned **no known malicious reports** for the IP address. This suggests that the alert was likely generated within a controlled lab environment rather than from an external threat.

However, in a real SOC investigation, analysts would continue monitoring the host system and review additional logs to confirm whether any malicious activity occurred.

## Threat Intelligence Summary 

Threat intelligence validation was performed using VirusTotal and AlienVault OTX to analyze the IP address associated with the PowerShell alert. The IP was cross-referenced with global threat intelligence databases. No malicious activity was reported, suggesting the alert is part of a controlled lab environment rather than a real attack.

## Conclusion

This task demonstrated the process of **SOC alert triage and IOC validation**. By analyzing alerts in Wazuh and validating indicators through threat intelligence platforms, analysts can quickly determine whether suspicious activity represents a real security threat or benign behavior within the network.
